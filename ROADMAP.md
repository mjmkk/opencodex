# Roadmap

This document outlines the planned direction for OpenCodex.
Items are organized by phase, not strict timeline.

## Near-term (Active Development)

- [ ] Android client
- [ ] File browser: in-app code editing and saving
- [ ] Multi-model support in chat (switch models per-thread)
- [ ] Push notification reliability improvements
- [ ] Session persistence across app restarts (terminal)

## Medium-term

- [ ] macOS native client (Catalyst or SwiftUI)
- [ ] Web client (browser-based interface)
- [ ] Plugin/extension system for custom tools
- [ ] Team/shared workspace support
- [ ] Offline mode with sync on reconnect

## Long-term / Exploratory

- [ ] Self-hosted backend one-click deployment (Docker Compose)
- [ ] Integration with other AI providers
- [ ] Voice input support

## Recently Completed

- [x] Terminal feature: WebSocket-based half-screen panel (iOS + backend)
- [x] File browser: tree view, syntax highlighting, file search
- [x] Thread import/export
- [x] Tailscale Serve integration
- [x] SSE event batching for performance
- [x] Bilingual documentation (English / 中文)
- [x] Full CI/CD pipeline (backend tests + iOS build/test)
- [x] Open-source readiness (Apache 2.0, contributing guides, security policy)

---

> Roadmap is subject to change. Community feedback and contributions welcome.
